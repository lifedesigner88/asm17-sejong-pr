## 3. Fix: Admin Seed Crash on Restart (Duplicate Email)

### What Changed
- `sync_admin_seed` in `auth/service.py`: existence check changed from `WHERE user_id = ?` to `WHERE user_id = ? OR email = ?`

### Why It Mattered
When `ADMIN_SEED_USER_ID` env var changed between deploys (e.g. `"admin"` → `"ADM001"`), the old row was not found by `user_id`, triggering an unconditional INSERT that violated the unique constraint on `email`. Backend crash-looped on every restart against Supabase. Checking by email as well makes the seed idempotent regardless of `user_id` changes.

## 7. Refactor — Remove 6-char String `user_id`, Use Auto-increment Integer PK

### What Changed
**Backend (`apps/backend/`):**
- `auth/models.py`: removed dual-column `id (Integer PK)` + `user_id (String 64)`; replaced with single `user_id: Integer` auto-increment PK
- `auth/schemas.py`: removed `NormalizedUserId`, `SignupUserIdValue` type aliases; removed `user_id` field from `SignupRequest`; changed `user_id: str → int` in `SessionResponse` and `UserResponse`
- `auth/service.py`: removed `ADMIN_SEED_USER_ID` env var; `sync_admin_seed` queries by email only; `create_user` no longer sets `user_id`; JWT subject uses `str(user.user_id)`; `get_current_user` casts to `int(user_id)`
- `persona/models.py`: `owner_user_id: String(64) FK("users.user_id")` → `user_id: Integer FK("users.user_id")`
- `capture/models.py`: FK changed from `users.id` → `users.user_id`
- `capture/service.py`: all `current_user.id` → `current_user.user_id`
- `capture/schemas.py`: `owner_user_id: str → int`
- `seed.py`: removed `DEMO_USER_ID`; finds user by email; `Persona(owner_user_id=...)` → `Persona(user_id=user.user_id, ...)`
- `.env.example`: removed `ADMIN_SEED_USER_ID`

**Frontend (`apps/frontend/`):**
- `auth/types.ts`: `user_id: number`, removed `generatedUserId`
- `auth/api.ts`: removed `user_id` from signup body; returns `signupEmail` only on success
- `auth/pages/signup-page.tsx`: removed 6-char ID card (generate/copy/shuffle UI), Step 2 ID display box, Step 3 ID confirmation screen
- `public/locales/en/auth.json` + `ko/auth.json`: removed `personaIdLabel`, `personaIdHint`, `copy`, `copied`, `shuffle`, `writeDown`; updated descriptions

### Why It Mattered
The 6-char client-generated user ID added friction: users had to write it down, copy it, and use it to log in. It was also a unique-collision risk. Replacing it with a DB auto-increment integer removes the ID card step from signup entirely — users sign up with email + PIN only. `user_id` is now an internal integer returned after signup, not a user-visible concept. The `users` table column rename from `id → user_id` (as sole PK) also simplifies all FK references.
