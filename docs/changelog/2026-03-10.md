# 2026-03-10

## 1) Local / Docker / Compose execution paths standardized
- local 개발 경로를 `pnpm dev` + `pnpm infra:down` 기준으로 고정했다.
- Dockerfile 검증 경로를 `pnpm docker`, `pnpm docker:logs`, `pnpm docker:down`으로 분리했다.
- Compose 데모는 `docker compose up`으로 바로 확인할 수 있게 유지했다.
- `infra:down`은 이름 그대로 `db`, `redis`만 정리하도록 수정했다.

## 2) Runtime and environment policy tightened for reproducibility
- Node `24.11.0`, Python `3.11.15`, Postgres `16.13-alpine`, Redis `7.4.7-alpine`로 patch 버전까지 고정했다.
- app 로컬 env는 `apps/*/.env.example`만 추적하고, 실제 `.env`는 실행 전에 자동 생성하도록 바꿨다.
- Compose 데모용 env는 루트 `compose.env` 하나로 정리하고 역할별 주석을 추가했다.

## 3) Docker build path stabilized
- frontend / backend Dockerfile을 추가하고 BuildKit 기반 빌드 경로를 정리했다.
- `pnpm docker`는 Nx stream 출력 + `docker build --progress=plain`으로 단계 로그를 추적할 수 있게 했다.
- frontend Dockerfile은 의존성 설치 레이어 캐시가 재사용되도록 `package.json` 중심으로 분리했다.
- Docker와 로컬이 `node_modules`, `.venv`, Vite cache를 공유하지 않도록 경로를 분리해 권한 충돌을 막았다.

## 4) Documentation and project guidance refresh
- 루트 `CHANGELOG.md`를 제거하고 `docs/changelog/YYYY-MM-DD.md` 구조로 전환
- README를 외부 학습자가 바로 실행/이해할 수 있는 형태로 전면 재구성
- AGENTS는 실제 운영 규칙, 실행 정책, env 정책 중심으로 다시 정리

## 5) Capture submission and frontend UI refinement
- capture draft는 `sessionStorage` 대신 브라우저 메모리 기반으로 바꾸고, review 단계에서 backend `POST /capture/jobs`로 실제 제출되도록 연결했다.
- capture 화면은 `Card`, `Badge` 같은 shadcn 스타일 공용 컴포넌트 기준으로 다시 정리했다.
- step 페이지의 반복 구조는 feature 내부 공통 shell / action 컴포넌트로 추출했다.
- 새 프론트 페이지는 `common/components`를 우선 조합하는 원칙을 문서에 고정했다.
- 전체 배경 톤은 한 단계만 눌러서 현재보다 덜 밝게 조정했다.
- 커밋 승인 이후에는 `커밋 -> 푸시 -> PR 생성`까지를 하나의 기본 루틴으로 문서에 고정했다.
- capture 제출물은 카드형 목록과 상세 화면으로 조회할 수 있게 확장하고, 상세 화면에서 삭제 가능하도록 구성했다.
- later admin 화면에서도 재사용할 수 있도록 `CaptureJobCard`, `CapturePayloadSummary` 같은 공용 capture 컴포넌트를 추가했다.
- backend 테스트는 이전 테스트 상태를 재사용하지 않고, 로그인/회원 생성 준비만 fixture로 공용화하는 방향으로 정리했다.
- `pwdlib`가 예전 해시를 못 읽을 때 startup이 죽지 않도록 `UnknownHashError`를 안전하게 처리하고, admin seed는 자동 재해시되도록 보강했다.
- 브랜치 정책은 `main` 고정이 아니라, 독립 작업은 `main`, 의존 작업은 이전 작업 브랜치를 부모로 하는 stacked branch 방식으로 정리했다.
- 새 PR을 올리기 전에는 이미 merge 된 브랜치를 로컬/원격 정리 대상으로 확인하는 정책을 추가했다.
